## üõ† **Architecture**

### **Pipeline:**
```
Text Input ‚Üí Parser ‚Üí SValue Tree ‚Üí Execution Engine ‚Üí JSON Output
```

### **Core Types:**
```fsharp
type SValue =
  | SNull | SBool | SString | SNumber     // Basic types
  | SArray | SObject                      // Collections
  | SRef of string list                   // $entity.id
  | SAction of string list                // [@rag:search]
  | SBinaryOp of SValue * string * SValue // $a*$b
```

### **Execution Flow:**
1. **Parse** text into SValue tree
2. **Find references** (`SRef`) ‚Üí call `skogcli config get`
3. **Find actions** (`SAction`) ‚Üí call `skogcli script run`
4. **Parse results** ‚Üí may contain more references/actions
5. **Recursive resolution** ‚Üí continue until concrete
6. **Output** final JSON with all semantics resolved
